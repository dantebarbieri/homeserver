# CLAUDE.md

This file provides guidance to Claude Code when working with the `nixos/` directory.

## Key Files

- `configuration.nix` — Single-file declarative system config (~520 lines). All services, packages, networking, and users defined here.
- `hardware-configuration.nix` — **Auto-generated by `nixos-generate-config`. Do not edit manually.**

`/etc/nixos` on the server is a symlink to this directory, so edits here are applied directly by `nixos-rebuild switch`.

## File Structure of `configuration.nix`

1. **`let` block** — Reusable values and scripts (`ntfyUrl`, `ntfyTopic`, alert scripts as `pkgs.writeShellScriptBin`)
2. **Boot/Loader** — systemd-boot, EFI, kernel packages
3. **Networking** — systemd-networkd (not NetworkManager), static IP, no global DHCP
4. **Storage** — LVM, RAID (mdadm), XFS on `/data`
5. **Users/Shell** — zsh default, zim framework, packages
6. **System packages** — `environment.systemPackages`
7. **Security** — SSH (key-only, port 28), sudo-rs (not classic sudo)
8. **Services** — Docker, smartd, systemd timers, NVIDIA GPU
9. **State version** — required by NixOS, do not change

## Common Patterns

### Systemd service + timer pair

```nix
systemd.services.my-service = {
  description = "...";
  after = [ "network-online.target" ];
  wants = [ "network-online.target" ];
  path = with pkgs; [ curl bash ];      # executables available in script
  serviceConfig = { Type = "oneshot"; };
  script = ''
    # shell script body
  '';
};

systemd.timers.my-service = {
  wantedBy = [ "timers.target" ];
  timerConfig = {
    OnCalendar = "*-*-* 04:00:00";
    Persistent = true;                   # run missed timers on boot
  };
};
```

### ntfy alerting

All monitoring scripts send alerts via curl to `${ntfyUrl}/${ntfyTopic}`:

```nix
${pkgs.curl}/bin/curl -s \
  -H "Title: ALERT on ${config.networking.hostName}" \
  -H "Priority: high" \
  -H "Tags: warning,computer" \
  -d "message body" \
  "${ntfyUrl}/${ntfyTopic}"
```

Priority levels: `urgent` (RAID failure), `high` (SMART warnings), `default`, `low` (info).

### Activation scripts

One-time setup tasks (e.g., deploy key generation):

```nix
system.activationScripts.name = lib.stringAfter [ "users" ] ''
  # runs on nixos-rebuild switch
'';
```

## Active Timers

| Timer | Schedule | Purpose |
|-------|----------|---------|
| `docker-compose-update` | Daily 04:00 (±5m) | Pull monorepo, update containers |
| `vdirsyncer-sync` | Every 15 min | Sync iCloud contacts via CardDAV |
| `drive-health-check` | Every 6 hours | RAID + LVM health → ntfy |
| `mdadm-scrub` | Weekly Sunday 02:00 | RAID parity scrub |

## Important Caveats

- Use full Nix store paths in scripts: `${pkgs.curl}/bin/curl`, not just `curl` (unless added to `path`)
- Services don't inherit shell aliases/functions — use `path` and `environment` explicitly
- The docker-compose-update service uses `git -c safe.directory=/srv/homeserver` because the repo is owned by `danteb` but the service runs as root
- `sudo-rs` replaces classic sudo — the Rust implementation

## Commands

```bash
sudo nixos-rebuild switch    # Apply changes

```
